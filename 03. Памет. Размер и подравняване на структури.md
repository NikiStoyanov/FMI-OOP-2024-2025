### Endianness
Това е начин за подреждане на **байтовете** в една процесорна дума.
Съществуват два вида Endianness:
- **Big-Endian** - подредбата се запазва;
- **Little-Endian** - байтовете се обръщат, но битовете във всеки байт се запазват, всички модерни процесори го използват.

За определяне на вида на подреждането най-лесно би било по следния начин: 
```c++
bool getAlignment()
{
	union A
	{
		int a = 1;
		char c;
	};
	
	A obj;
	
	return obj.c == 1;
}
```
Тогава подравняването е:
- **Little-Endian**, ако функцията връща `true`;
- **Big-Endian**, ако функцията връща `false`;
### Размер на структури
```c++
struct A
{
	int x; // 4B
	char y; // 1B
};
```
Тук очакваното заделено място е $5B$, само че за улеснение на процесорите се използва т. нар. **подравняване (padding)**, което представлява заделяне на допълнителна памет.
- големината на празна структура е $1B$;
- **sizeof(A)** - връща размера на А;
### Подравняване на структури
- Подравнява се по най-голямата по размер променлива от примитивен тип;
- Всяка променлива се разполага на адрес, който се дели на големината ѝ.
- Структурата се разполага на адрес, който се дели на целия ѝ размер след подравняване.

>Винаги подреждаме полетата по големина, без значение дали възходящо или низходящо с цел да минимализираме padding-а.

- **alignment** - наричаме променливата, по която подравняваме.
- **alignof(A)** - връща размера на подравняването.

```c++
struct D
{
	char c; // 1B
	float f; // padding 3B + 4B
	int* ptr; // 8B
};

struct E
{
	uint16_t a; // 2B
	int f; // padding 2 + 4B
	D obj; // 16B
};

int main()
{
	std::cout << alignof(D) << std::endl; // 8
	std::cout << sizeof(D) << std::endl; // 16
	
	std::cout << alignof(E) << std::endl; // 8
	std::cout << sizeof(E) << std::endl; // 24
}
```
Най-голямата по размер променлива в текущата и във вложените структури определя подравняването на структурата.
